Configuration Files¶

Basics¶

Configuration files in this project are INI files as supported by the configparser Python module. Refer to that documentation for all the detailed features of the format.

The files consist of sections, delimited by a [section] header and containing name=value properties. Leading whitespace is removed from values. Lines starting with # or ; are treated as comments.

#       Y: 4 digit year
#       m: 2 digit month
#       b: 3 character month (Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)
#       d: 2 digit day
#       y: 2 digit year
#       j: 3 digit day of year
#       H: 2 digit hour (24-hour time)
#       M: 2 digit minute
#       S: 2 digit second
#       MILLI: 3 digit millisecond
#       MICRO: 3 digit microsecond

mission¶

This section defines the mission; this configuration only supports one mission per database. The keys map directly to the fields in the mission table:

  • rootdir

  • mission_name

  • incoming_dir

satellite¶

Although a database may contain multiple satellites, products can only be defined within a single config file for a single satellite. (Multiple config files can be used to add multiple satellites.)

The satellite_name key specified the name of the satellite as in satellite.satellite_name; the instrument and products in this file will all be associated with the satellite of this name.

instrument¶

As with satellite, this section specifies, via instrument_name, the instrument (instrument.instrument_name) with which all the products in this file will be associated.

process¶

Each section starting with process_ defines a process. An entry in the process table will be added for each section; the keys map to columns of the table:

process_name

process_name

output_product

output_product. This is specified in the config file as a name, and mapped to the product_id. If the product exists in the same config file, it must be the name of the product section, including the product_. If the product is already in the database, it must be specified as the product_name. (Making this more flexible is planned.)

output_timebase

output_timebase

extra_params

extra_params

Keys starting with optional_input or required_input create entries in the productprocesslink table. They must be named sequentially, e.g.:

  • required_input1

  • required_input2

  • optional_input1

etc.

The values of each entry are ordinarily the name of a product. As with the output product, if the product exists in the same config file, it must be the name of the product section, including the product_. If the product is already in the database, it must be specified as the product_name.

If the key starts with optional, the optional column is True; otherwise it is False.

To specify the inclusion of previous or following days of input data (i.e. to set yesterday and tomorrow, use a different format, of ("name", days_before, days_after). For example, to specify the product from section product_level2 as a required input, and use one day prior to the current day as input and no addition days after, use:

required_input1 = ("product_level2", 1, 0)

The product name is only in quotes if using this form.

# sample config file

mission = ~/RBSP_MAGEIS.sqlite
outformat = pdf
filename_format = MagEIS_L3_Coverage_{TODAY}
startdate = 20120901

###############################################
# Plots
###############################################
[panel]
# in the panel section we define what will be plotted
# N keys pf plotN define subplots
# daysperplot gives the days per plot that will be on each page
plot1 = plot1
plot2 = plot2
daysperplot = 60
title = MagEIS L3 Coverage
preset = green
missing = red

makeLatestSymlinks.py¶

[isois]
# Directory containing the data files
sourcedir = ~/dbp_py3/data/ISOIS/level1/
# Directory to make the symlinks in
destdir = ~/tmp/
# First date to link
startdate = 2010-01-01
# Last date to link
enddate = 2021-01-01
# Number of days before present not to link (e.g. to keep internal-only)
deltadays = 60
# glob for files to match
filter = psp_isois_l1-sc-hk_*.cdf
# Link directories as well as files
linkdirs = True
# Mode to use when making output directory
outmode = 775
# Do not limit based on date (i.e., ignore date options; they're still required)
nodate = False
